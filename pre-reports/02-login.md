# Вход в систему (Login)

**Маршрут:** `/auth/login`
**Доступ:** только для неавторизованных пользователей

## Описание

Центрированная карточка с формой авторизации на сером фоне.

## Элементы интерфейса

### Заголовок
- Название: **«Молочная ферма»**
- Подзаголовок: «Система управления животноводством»

### Форма входа
- **Логин** — текстовое поле с плейсхолдером «Введите логин»
- **Пароль** — поле типа password с плейсхолдером «Введите пароль»
- **Кнопка «Войти»** — полная ширина, синяя, при загрузке показывает «Входим...»
- **Сообщения об ошибках** — красный блок над формой при неудачной попытке

### Примечание
- Текст внизу: «Забыли пароль? Обратитесь к администратору системы.»

## Валидация

- Оба поля обязательны для заполнения
- Валидация срабатывает при отправке формы

## Что происходит при входе

1. Отправляется POST-запрос `/auth/login` с логином и паролем
2. Сервер возвращает JWT-токены (access + refresh) в HttpOnly cookies
3. Сохраняются имя пользователя, роль и флаг `mustChangePassword`
4. Загружаются пользовательские настройки (тема, размер страницы)
5. Перенаправление на главную страницу (`/`)
